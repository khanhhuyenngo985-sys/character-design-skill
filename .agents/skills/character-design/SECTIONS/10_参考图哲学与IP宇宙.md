## 参考图哲学

### 核心原则

> **给AI一张图，胜过写一万字提示词。**

### AI如何解读参考图

AI 视频/图片模型接收参考图后，会从中"推演"出：

```
□ 角色的体型与动作规律
   - 老人的驼背重心
   - 壮汉的重心分布
   - 儿童的蹦跳姿态

□ 角色的声音气质
   - 某些平台的模型会根据外形生成匹配声音
   - 冷酷角色 → 低沉声音
   - 活泼角色 → 明快声音

□ 角色的情绪演技风格
   - 慵懒 vs 紧张 vs 冷酷
   - 表情变化节奏
   - 肢体语言模式

□ 角色的行走姿势与肢体语言
   - 步幅大小
   - 手臂摆动
   - 头部位置
```

---

### 参考图的类型

#### 1. 标准三视图

```
□ 正脸图（正面头像/上半身）
   - 看清面部特征
   - 看清发型、额头
   - 建立基础印象

□ 侧脸图（90°侧脸）
   - 看清颧骨、下巴轮廓
   - 看清鼻子高度
   - 侧脸是角色辨识关键

□ 全身图（站立姿态）
   - 看清体型比例
   - 看清重心位置
   - 看清服装整体
```

#### 2. 姿态参考图

```
□ 日常态（放松）
   - 重心居中
   - 肌肉松弛
   - 自然站姿

□ 紧张态（战斗/压力）
   - 重心前倾
   - 肌肉紧绷
   - 防御姿势

□ 亲密态（信任/开放）
   - 身体微倾
   - 开放姿态
   - 面部放松
```

#### 3. 表情参考图

```
□ 八种核心情绪
   - 愤怒/悲伤/恐惧/惊讶
   - 厌恶/轻蔑/喜悦/平静

□ 组合表情
   - 复杂情绪
   - 微表情
```

#### 4. 服装参考图

```
□ 主场服装
   - 主要场景穿着
   - 日常态

□ 特殊服装
   - 战斗服/正装
   - 特殊状态

□ 配件细节
   - 珠宝/武器/道具
   - 局部特写
```

---

### 参考图质量标准

```
好参考图：
□ 高清（至少1024x1024）
□ 光线清晰（明暗可辨）
□ 正面角色（不被遮挡）
□ 姿态标准（不扭曲）
□ 色彩准确（不偏色）
□ 背景干净（不干扰）

坏参考图：
□ 低清/模糊
□ 光线过曝/欠曝
□ 角色被遮挡
□ 姿态扭曲
□ 色彩失真
□ 背景杂乱
```

---

### 参考图使用技巧

#### 1. 参考链建立

```
前一镜输出图 → 下一镜参考 + 三视图
```

**好处**：
- 保持连续性
- 减少累积误差
- AI更容易"记住"角色

#### 2. 多参考图组合

```
□ 正面图：锁定面部
□ 侧脸图：锁定轮廓
□ 姿态图：锁定身体语言
□ 服装图：锁定材质
```

#### 3. 参考图权重

```
□ 强制参考（0.8-1.0）
   - 面部特写
   - 角色一致性问题

□ 中等参考（0.5-0.7）
   - 姿态参考
   - 服装参考

□ 弱参考（0.3-0.5）
   - 风格参考
   - 光影参考
```

## 设定稿的作用

### 设定稿 = 写给AI的角色说明书

```
一张好的设定稿能让AI记住：
□ 体型
□ 颜色
□ 姿态
□ 服装纹理
□ 情绪质感
□ 道具尺度
```

### 设定稿应该一次性告诉AI

```
1. 角色性格气质（通过姿态和神情体现）
2. 专属色彩标识
3. 至少一个标志性姿态
4. 服装材质层次感
5. 道具的位置关系与尺度感
```

## IP宇宙思维

### 旧认知 vs 新认知

| 旧认知 | 新认知 |
|--------|--------|
| 做剧 = 最终产品 | 做剧 = 世界观前期容器 |
| 写小说 = 写小说 | 写小说 = 为游戏化/动画化做前期 |
| 成片 = 终点 | 成片 = 动态分镜，二创的起点 |
| 观众 = 消费者 | 观众 = 潜在二创者/共建者 |

---

### AI时代的制作平权

```
AI时代制作工具趋于平等
（人人都能用同款模型生成视频）

最终拉开差距的只有前期：
□ 世界观是否有延展性
□ 角色是否有被二创的空间
□ 设定是否足够开放和差异化
□ 道具是否能成为文化符号
```

---

### 角色包容性设计

**今天的观众不再是被动接受者**：
- 截图者
- 表情包制作者
- 二创者
- 同人作者

**设计原则**：
```
□ 任何配角，哪怕连台词都没有
   都可能被观众拉出去开创新的叙事

□ 角色要具备多重解读空间
   不同性格的人都能在某个角色上找到投射点

□ 角色宇宙（Character Universe）
   就是这样自然生长出来的
```

---

### 互动创作生态的终极闭环

```
创作者做前期（世界观/角色设计）
           ↓
成片上线（= 发布可互动的世界观原型）
           ↓
观众截图、做表情包、写同人、二创视频
           ↓
评论区成为互动剧场
           ↓
角色IP自然裂变扩散
           ↓
商业价值（联名/周边/授权）
```

### 问题→解决

| 问题 | 解决 |
|------|------|
| AI角色总变脸 | 参考图+DNA完整+每镜检查 |
| 角色融进背景 | 色彩对比+强光源+色彩锚点 |
| 多角色分不清 | 群像色彩隔离+位置固定 |
| 姿态总是不对 | 精确描述重心/四肢/脊椎三要素 |
| 表情传达不了情绪 | 情绪-表情-姿态联动描述 |
| 镜头间角色不一致 | 三层锚定+Shot-to-Shot参考链 |
| 观众记不住角色 | 强化色彩锚点+姿态签名 |
| 角色太完美 | 添加缺陷/矛盾/争议 |
| 配角没记忆点 | 给配角设计独特DNA+道具 |

## 设定稿检查清单

### 面部

```
□ 脸型描述清晰（圆/方/椭圆/瓜子）
□ 颧骨高/低
□ 眉形（粗/细/弯/直）
□ 眼形（大/小/凤眼/圆眼）
□ 鼻形（高/低/鹰钩/蒜头）
□ 嘴形（大/小/薄/厚）
□ 下巴（尖/方/圆）
□ 特殊标记（疤/痣/纹身）
```

### 体型

```
□ 身高（具体数值）
□ 肩宽（宽/窄）
□ 体型（瘦/正常/壮/胖）
□ 重心位置（高/中/低）
```

### 色彩

```
□ 主色（具体颜色+饱和度）
□ 辅色（具体颜色）
□ 点缀色（具体颜色）
□ 色彩锚点（高饱和局部细节）
```

### 姿态

```
□ 站姿（挺直/驼背/重心偏向）
□ 坐姿（标准/二郎腿/瘫坐）
□ 手部习惯（插兜/抱胸/自然下垂）
□ 标志性动作（具体描述）
```

### 服装

```
□ 主服装（颜色+材质+款式）
□ 配件（项链/耳环/眼镜/手套）
□ 鞋子（颜色+款式）
□ 特殊道具（武器/工具/宠物）
```

## IP衍生检查

### 可衍生性评估

```
□ 视觉辨识度
   - 角色剪影是否独特？
   - 色彩是否让人一眼记住？

□ 性格辨识度
   - 是否有让人难忘的性格标签？
   - 是否有经典台词/动作？

□ 情感共鸣度
   - 观众是否能代入角色？
   - 是否有让人心疼/喜欢的点？

□ 二创空间
   - 是否有未完结的故事线？
   - 是否有可以发展的关系？
   - 是否有可以探索的背景？

□ 商业价值
   - 是否有周边潜力（手办/服装）？
   - 是否有联名价值（品牌合作）？
   - 是否有授权价值（游戏/动画）？
```

## 案例：从设定到IP

### Wednesday Adams案例

```
设定：
□ 黑白服装 = 独特视觉
□ 断手 = 独特道具
□ 反社会人格 = 争议性格
□ 与环境隔绝 = 孤独感

二创：
□ 表情包（Wednesday凝视/断手）
□ 同人小说（与Enid的友情）
□ 时尚灵感（暗黑哥特风）
□ 话题讨论（心理健康）

商业：
□ 联名（MAC唇膏/周三色）
□ 周边（断手玩具/哥特娃娃）
□ 授权（游戏/舞台剧）
```
