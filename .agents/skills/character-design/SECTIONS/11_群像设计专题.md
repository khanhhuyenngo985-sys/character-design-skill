# 群像设计：角色互补系统

> 角色不是孤立的，角色只有在关系中才有意义。群像设计的核心是角色互补。

---

## 核心原则

**群像设计的目的**：让每个角色在其他角色的映衬下，变得更鲜明、更独特。

**互补公式**：
```
角色A的某特征 + 角色B的相反特征 = 两者都被强化
```

**反面例子**：
```
两个都是"温柔善良"的角色 → 群像扁平、无张力
两个都是"冷酷杀手" → 观众分不清谁是谁
```

---

## 互补维度

### 一、性格互补

| 维度 | 角色A | 角色B | 效果 |
|------|-------|-------|------|
| 行动vs思考 | 冲动派 | 谋略派 | 冲突与平衡 |
| 开放vs封闭 | 外向活泼 | 内向沉默 | 节奏变化 |
| 理想vs现实 | 追梦者 | 务实者 | 矛盾与妥协 |
| 强硬vs柔软 | 铁腕者 | 温和者 | 权力张力 |

**案例：功夫**
```
星仔（Hero/天真） + 酱爆（喜剧/Trickster）= 节奏互补
包租婆（Threshold Guardian/强硬） + 裁缝（Ally/配合）= 角色关系清晰
```

### 二、色彩互补

**色相环互补关系**：

| 角色A主色 | 角色B主色 | 色相差 | 效果 |
|-----------|-----------|--------|------|
| 红 | 青/蓝 | 180° | 最强对比 |
| 橙 | 蓝 | 150° | 强对比 |
| 黄 | 紫 | 120° | 中对比 |
| 暖色 | 冷色 | 任意 | 温度对比 |

**实战方案**：
```
方案1：高饱和 + 低饱和
- 角色A：高饱和红色系（热情/主动）
- 角色B：低饱和灰色系（冷淡/被动）

方案2：暖色 + 冷色
- 角色A：暖色调（橙/黄/红）
- 角色B：冷色调（蓝/青/紫）

方案3：主色 + 点缀色对调
- 角色A：红色主色 + 蓝色点缀
- 角色B：蓝色主色 + 红色点缀
```

**案例：赛博朋克边缘跑者**
```
David：黄夹克（暖色/人性）
Lucy：蓝色系（冷色/神秘）
Rebecca：绿红配（极端对比/危险）
```

### 三、体型互补

| 体型类型 | 特征 | 适合原型 |
|----------|------|----------|
| 高+瘦 | 修长、敏捷 | Hero、Mentor、Ally |
| 高+壮 | 压迫、力量 | Shadow、Threshold Guardian |
| 矮+瘦 | 灵活、狡猾 | Trickster、Shapeshifter |
| 矮+壮 | 敦实、可靠 | Ally、Threshold Guardian |
| 正常 | 普通、融入 | Herald、Shapeshifter |

**群像体型分配原则**：
```
□ 每个体型类型只出现一次
□ 主角和主要对手体型要有明显差异
□ 避免两个"正常体型"角色同屏
□ 体型与原型匹配（杀手不该是矮胖）
```

### 四、姿态互补

| 角色A姿态 | 角色B姿态 | 效果 |
|-----------|-----------|------|
| 驼背 | 挺直 | 自信vs压抑 |
| 张开 | 收缩 | 开放vs封闭 |
| 前倾 | 后仰 | 主动vs被动 |
| 静止 | 运动 | 沉稳vs活泼 |

**群像姿态分配**：
```
□ 驼背/挺拔/展开/收缩 → 各有分配
□ 原型与姿态匹配：
  - Shadow（威胁）→ 俯视/收缩/静止
  - Hero（成长）→ 挺拔但不稳定
  - Ally（支持）→ 舒展/开放
  - Trickster（打破）→ 不规则/运动中
```

### 五、原型互补

**经典互补配对**：

| 配对 | 角色A（原型） | 角色B（原型） | 叙事功能 |
|------|---------------|---------------|----------|
| 对手 | Hero | Shadow | 冲突核心 |
| 导师 | Mentor | Hero | 成长驱动 |
| 伙伴 | Ally | Hero | 支持系统 |
| 测试者 | Threshold Guardian | Hero | 考验 |
| 变数 | Shapeshifter | 任意 | 悬念 |
| 打破者 | Trickster | 任意 | 意外 |

**原型叠加增强互补**：
```
Hero + Shadow = 悲剧英雄（与自己的阴影战斗）
Hero + Mentor = 牺牲者（用死亡教育他人）
Shadow + Ally = 反转角色（表面敌人实际帮助主角）
Mentor + Shadow = 反差导师（平时普通实为高手）
```

## 群像关系设计

### 六种核心关系

**1. 对立关系**
```
关系：Hero vs Shadow
特征：直接冲突、目标相反、最终对决
视觉：左右对峙、视线交汇、明暗对比
```

**2. 成长关系**
```
关系：Hero vs Mentor
特征：引导与被引导、知识传递、最终离别
视觉：高低位置（坐vs站）、视线（仰望vs俯视）
```

**3. 支持关系**
```
关系：Hero vs Ally
特征：并肩作战、相互信任、不离不弃
视觉：并肩站立、同一方向、色彩协调
```

**4. 考验关系**
```
关系：Hero vs Threshold Guardian
特征：必须通过才能前进、有明确规则
视觉：阻挡姿态、入口/门口位置
```

**5. 悬念关系**
```
关系：Hero vs Shapeshifter
特征：身份不确定、可能反转
视觉：模糊角度、阴影遮挡、半遮面
```

**6. 喜剧关系**
```
关系：任意 vs Trickster
特征：打破严肃、意外、规则破坏
视觉：不规则姿态、动作定格、表情反差
```

### 群像视线优先级

**规则**：观众视线顺序 = 角色重要性顺序

```
视线设计原则：
□ 最重要的角色被看最多
□ 对立关系时视线交汇
□ 配角看向主角 = 配角是次要
□ 主角看向画面外 = 引导观众好奇
□ 避免无意义的"互看"
```

**视线冲突解决**：
```
问题：两个角色都想被关注
解决：
1. 使用景别差异（近景vs远景）
2. 使用位置差异（左1/3vs右1/3）
3. 使用色彩差异（饱和vs低饱和）
4. 使用光影差异（亮vs暗）
```

## 群像色彩隔离

### 色彩隔离原则

**核心**：每个角色有专属色彩区域，远看可区分。

**检查公式**：
```
角色A主色 ≠ 角色B主色 ± 30°色相
角色A饱和度 ≠ 角色B饱和度 ± 20%
```

### 色彩分配策略

**方案1：象限分配**
```
色彩冷暖（纵轴）+ 饱和高低（横轴）

        高饱和    低饱和
  暖色   A角色     B角色
  冷色   C角色     D角色
```

**方案2：主色分配**
```
角色A：红色系（0-30°）
角色B：蓝色系（180-210°）
角色C：黄色系（45-75°）
角色D：紫色系（270-300°）
```

**方案3：点缀色分配**
```
角色A：灰色主色 + 红色点缀
角色B：灰色主色 + 蓝色点缀
角色C：灰色主色 + 黄色点缀
（灰色主色统一，点缀色区分）
```

### 高危场景色彩方案

**夜间/昏暗场景**：
```
问题：角色色彩被黑暗吞噬
解决：
1. 每个角色有专属光源色（霓虹/路灯）
2. 使用高饱和点缀色（围巾/耳钉/鞋）
3. 角色间色彩差异最大化
```

**自然场景**：
```
问题：角色肤色融入环境
解决：
1. 服装饱和度 > 环境饱和度
2. 使用对比色点缀（绿+红、蓝+橙）
3. 侧光分离主体背景
```

## 群像设计检查清单

### 互补性检查

```
【性格互补】
□ 角色A和角色B的性格差异是否明显？
□ 差异是否产生冲突或张力？
□ 是否有"重复"的角色（性格相似无法区分）？

【色彩互补】
□ 每个角色的主色是否在色相环上不同？
□ 饱和度是否有足够差异？
□ 是否有色彩锚点（高饱和局部细节）？

【体型互补】
□ 体型类型是否各不相同（高/矮/胖/瘦）？
□ 体型是否与原型匹配？
□ 是否避免了"两个普通人站一起"？

【姿态互补】
□ 站姿是否有差异（挺直/驼背/张开/收缩）？
□ 姿态是否反映角色性格？
□ 原型与姿态是否匹配？

【原型互补】
□ 原型叠加是否 ≥ 2？
□ 是否有对立原型（Hero vs Shadow）？
□ 原型关系是否推动叙事？
```

### 一致性检查

```
【参考图】
□ 每个角色都有三件套（正脸/侧脸/全身）？
□ 参考图是否同角色同服装？

【DNA标签】
□ 每个角色的DNA是否完整？
□ 多人场景是否写明了位置区分？

【色彩锚点】
□ 主色是否每次都声明？
□ 点缀色是否每次都提及？

【位置关系】
□ 角色位置是否与前后镜头一致？
□ 是否避免了越轴问题？
```

### 视觉权重检查

```
【主角突出】
□ 主角是否占据视觉焦点位置？
□ 主角色彩是否最突出？
□ 主角是否获得最多视线？

【配角分工】
□ 每个配角是否有独特记忆点？
□ 配角是否服务主角而非抢戏？
□ 配角之间是否可区分？

【群像平衡】
□ 视觉重量是否平衡（不能一边重一边轻）？
□ 色彩分布是否均衡？
□ 位置关系是否稳定？
```

## 案例：外卖骑手老林群像

### 角色关系设计

```
老林（Hero/Shadow）↔ 小赵（Ally/Trickster）
     ↓
老板娘（Threshold Guardian/Shadow）
```

### 互补设计

| 维度 | 老林 | 小赵 | 老板娘 |
|------|------|------|--------|
| 性格 | 沉默/压抑 | 开朗/话多 | 强硬/不耐烦 |
| 色彩 | 深蓝（冷） | 黄色（暖） | 红色（暖） |
| 体型 | 偏瘦/175cm | 壮/170cm | 丰满 |
| 姿态 | 驼背/重心偏 | 舒展/重心中 | 叉腰/重心稳 |
| 原型 | Hero+Shadow | Ally+Trickster | Threshold Guardian |

### 色彩隔离验证

```
深蓝 + 黄 + 红 = 三个完全不同色相
色相差：蓝↔黄≈120°，黄↔红≈30°，蓝↔红≈150°
→ 远看可区分 ✓
```

### Blocking设计

```
老林（左1/3）↔ 老板娘（右1/3）
     ↓ 视线
     小赵（后景/辅助）
```

## 群像设计质量分级

### A级（可直接用于生产）

```
□ 每个角色都有 ≥ 3个互补维度
□ 色彩在色相环上差异 ≥ 60°
□ 体型类型各不相同
□ 原型配对有叙事功能
□ 无"重复角色"
```

### B级（需要小幅调整）

```
□ 有2-3个互补维度
□ 色彩差异 30-60°
□ 体型大部分不同
□ 有1个"重复角色"需要修改
```

### C级（需要重新设计）

```
□ 互补维度 < 2
□ 色彩过于接近
□ 多个"重复角色"
□ 原型配对混乱
```
