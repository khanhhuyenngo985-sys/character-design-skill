# 进化建议生成模板

## 输入信息

```
技能: {skill_name}
技能ID: {skill_id}
进化模式: {mode}  (REFINE | BRANCH | CAPTURE)
触发原因: {trigger_reason}
当前指标:
  - 调用次数: {applied_count}
  - 成功率: {success_rate}%
  - 失败率: {failure_rate}%
  - 平均质量: {avg_quality}/5
  - 捕获模式: {pattern_count}
```

## 进化执行指南

根据进化模式，按照以下步骤执行：

### REFINE 模式（优化）

当成功率下降或失败率上升时触发：

1. **分析失败案例**
   - 提取失败记录中的共同模式
   - 识别失败的根本原因（提示词、环境、参数？）

2. **生成优化建议**
   ```
   建议优化点:
   - <具体参数调整>
   - <提示词修改>
   - <新增约束条件>
   ```

3. **制定变更计划**
   ```
   需要修改的文件:
   - skills/{skill_id}/SKILL.md
   - skills/{skill_id}/prompts/xxx.md (如适用)

   具体变更:
   - 第XX行: <原文> → <新文>
   - 新增第XX行: <内容>
   ```

### BRANCH 模式（分支）

当发现新场景/风格/平台组合时触发：

1. **识别新场景**
   ```
   新场景: <场景描述>
   适用性: <为何需要新分支>
   差异点: <与主技能的参数差异>
   ```

2. **创建分支技能**
   ```
   新技能名: {skill_id}-{variant}
   复制自: {skill_id}
   需要调整的参数:
   - <参数1>: <调整值>
   - <参数2>: <调整值>
   ```

3. **更新主技能文档**
   ```
   在 SKILL.md 中添加 BRANCH 说明:
   ## 分支版本
   - {skill_id}-<variant>: <场景描述>
   ```

### CAPTURE 模式（捕获）

当3+案例出现共同成功模式时触发：

1. **分析共同模式**
   ```
   共同模式:
   - 模式A: <描述> (出现X次)
   - 模式B: <描述> (出现Y次)
   - 模式C: <描述> (出现Z次)
   ```

2. **提炼核心特征**
   ```
   核心特征:
   - <特征1>: 来自<模式>
   - <特征2>: 来自<模式>
   ```

3. **更新技能**
   ```
   在 skills/{skill_id}/SKILL.md 中新增:
   ## 成功模式库
   <提炼的模式>

   或创建新技能:
   skills/{new_skill_id}/SKILL.md
   ```

---

## 变更确认清单

执行变更前，必须确认：

- [ ] 变更有明确的分析依据（非凭空猜测）
- [ ] 变更不会破坏现有功能的兼容性
- [ ] 变更经过小样本测试验证
- [ ] 变更内容已记录到 evolution-log.md
- [ ] 相关技能的文档已同步更新

## 变更记录模板

```markdown
### {date} - {mode} #{version}

**技能**: {skill_id}
**触发**: {trigger_reason}

**变更内容**:
{详细描述}

**影响评估**:
- 向后兼容: {是/否}
- 需同步更新: {相关技能}
- 风险等级: {低/中/高}

**确认**: ⬜ 待人工确认
```
