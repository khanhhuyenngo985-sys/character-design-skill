# Brief Parser

## TL;DR
```
客户Brief输入 → 竞品分析(可选) → 提取已知信息 → 翻译模糊语言 → 补全缺失字段 → 输出2-3个方向 → 用户选择 → 完整创作简报
```

## 启动检查点

激活后先确认：
1. **Brief来源**：文字/链接/图片/混合？
2. **项目类型**：广告/品牌片/社交内容/活动视频？
3. **是否需要竞品分析**：客户有指定行业或竞品？

> 确认后进入 Step 0.5（竞品分析）或直接 Step 1。

---

Parse messy, mixed-format client briefs into structured creative direction options and a complete creative brief.

## When to Use

- Client sends a brief via text, links, images, reference ads, or a mix of all
- Brief contains vague language ("高端大气上档次") that needs translation
- Client provides incomplete information (missing audience, timeline, budget)
- Team needs creative direction options to choose from before starting
- Project is any type: commercial ads, brand films, social content, event videos

## Input

Accept any of:
- Raw text (WeChat messages, email, Slack, etc.)
- URLs and links (reference ads, competitor ads, inspiration)
- Image descriptions (reference screenshots, mood board images)
- Mix of all the above

## Processing Pipeline

### Step 0.5: 竞品视觉分析（可选）

**触发条件**：客户指定了行业或竞争对手 → 执行；否则跳过直接进入 Step 1。

```
输入：行业名称 or 竞品名称
输出：竞品视觉分析摘要（2-3句话）+ 差异化定位建议

执行步骤：
1. 搜索同行业3-5个成功广告案例（小红书/抖音/B站/微博）
2. 提取每个案例：色调/光影/节奏/叙事结构
3. 对比白梦客美学系统，识别差异化定位
```

### Step 1: Extract Known Information

Identify what the client has already specified:

| 字段 | 来源 |
|------|------|
| 投放渠道 | 客户明确说明的渠道 |
| 交付物 | 客户明确说明的格式/时长/数量 |
| 预算 | 客户明确说出的数字或范围 |
| 周期 | 客户明确的截止日期或时长 |
| 目标受众 | 客户提供的描述 |
| 品牌调性 | 客户用过的形容词 |
| 核心信息 | 客户说过的产品/服务卖点 |
| 视觉风格 | 客户提供的参考或描述 |
| 时长 | 客户说明的秒数/分钟数 |

### Step 2: Translate Vague Language

Convert vague client language into professional creative terms:

| 客户说的 | 专业翻译 |
|---------|---------|
| 高端大气上档次 | 克制优雅、黑白灰主调、大量负空间、长镜头凝视 |
| 年轻人喜欢的 | Z世代审美、潮酷/亚文化、碎片化节奏、饱和色彩 |
| 有电影感 | 宽银幕构图、胶片颗粒感、长镜头叙事、侧逆光 |
| 中国风 | 东方留白、水墨意境、大地色+苍青、极简构图 |
| 国际范儿 | 极简设计、自然光、黑白灰、无多余装饰 |
| 有质感 | 16mm胶片、过曝1/3档、单一光源、材质特写 |
| 温暖的感觉 | 暖色调（琥珀色主光）、自然光散射、慢节奏 |
| 科技感 | 冷色调（蓝/灰）、CG动画元素、干净线条、快速剪辑 |
| 接地气 | 真实场景、非职业演员、自然光、口语化文案 |

### Step 3: Fill in the Gaps

For missing required information, professional defaults based on what IS known:

| 缺失字段 | 推断逻辑 |
|---------|---------|
| 目标受众 | 根据产品/服务类型推断，并注明"需客户确认" |
| 品牌调性 | 根据预算和渠道推断（高定线 vs 通用线） |
| 核心信息 | 从产品卖点反推，目标一句话传播主张 |
| 视觉风格 | 根据品牌调性匹配白梦客美学系统 |
| 叙事结构 | 根据时长和渠道决定（15秒短平快 vs 2-3分钟故事） |

### Step 4: Generate Creative Direction Options

**输出 2-3 个完整迷你简报**，每个包含：

```
## 方向X：[方向名称]

目标受众：[具体人群描述，2-3句话]
品牌调性：[2-4个关键词]
核心信息：[一句话传播主张]
视觉风格：[具体描述 + 色彩/光线/构图关键词]
叙事结构：[开头X秒/中段X秒/结尾X秒]
声音方向：[BGM风格 + 旁白调性 + 音效要求]
投放渠道：[对应这个方向的渠道 + 内容适配建议]
预估时长：[X秒 或 X分钟]
预估周期：[X天 或 X周]
预估成本：[低/中/高 — 基于所需制作复杂度]
```

**方向差异原则：**
- 方向之间必须有明显区别（不同叙事结构、不同视觉语言、不同情绪基调）
- 避免三个方向只有文案措辞不同的假差异
- 其中一个方向应尽量靠近白梦客美学系统（高定线或通用线）

## HKR 选题质检法

在评估创意方向时，使用 HKR 法判断选题质量：

| 维度 | 问题 | 检查 |
|------|------|------|
| **H (Happy)** | 足够有趣、有悬念？标题和开头能让人好奇想点开？ | [ ] |
| **K (Knowledge)** | 有信息量？看完能学到新东西？ | [ ] |
| **R (Resonance)** | 能戳中情绪？让人"对对对我也这么想"？ | [ ] |

**评级标准：**
- S级选题：H+K+R 三项兼备
- 及格选题：至少占两项
- 不及格选题：只占一项或零项 → 需重新审视方向

**选题来源参考：**
- X（Twitter）：抓一手AI资讯
- Reddit：r/artificial、r/ChatGPT 等（100+upvote算火）
- 小红书：大众舆论场里AI话题焦点
- 即刻：AI博主、产品创意、行业洞察
- 微博/抖音/B站：热搜话题（至少两三个平台同时火才算破圈）

## 选题来源与素材积累

**选题交集模型：**
一个好选题 = 你的专业领域 + 读者的普遍兴趣 + 当下的时间节点

光是专业，可能曲高和寡。光是读者兴趣，可能流于肤浅。光是蹭热点，可能昙花一现。三者结合才能找到最精准的引爆点。

**角色代入法（发散思考）：**
- "一个很忙的普通用户"：能帮我省时间/解决具体麻烦吗？
- "一个爱玩的朋友"：够有趣吗？能让我产生"卧槽还能这样？"的惊叹？
- "一个焦虑的学习者"：能让我学到新东西、有新角度吗？

## Quality Gates

输出前检查：

- [ ] 所有"客户提供"的信息有明确来源标注
- [ ] 所有"推断"的信息有[待确认]标注
- [ ] 创意方向之间有真实差异，不是文案措辞差异
- [ ] 视觉风格描述具体到可以直接生成提示词
- [ ] 预算和周期基于已知信息合理估算
- [ ] 至少一个方向适配白梦客美学系统
- [ ] **HKR选题质检**：每个方向都通过了 H/K/R 至少两项

## Notes

- 始终使用 humanize-dialogue 技能处理所有文案/台词部分
- 参考白梦客美学系统（高定线或通用线）为视觉方向打底
- 模糊信息翻译时给出2个可能方向并说明原因
- 周期估算考虑AI生成的时间（比实拍短但需迭代）
- **先判断选题是否值得做，再进入创意方向阶段**
